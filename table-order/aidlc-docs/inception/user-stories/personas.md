# 테이블오더 서비스 - 사용자 페르소나

---

## Persona 1: 고객 (Customer)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 페르소나) |
| **역할** | 매장 방문 고객 |
| **연령대** | 20~60대 (폭넓은 연령층) |
| **기술 수준** | 초급~중급 (스마트폰/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 내 테이블에 설치된 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 대기 시간 없이 빠르게 주문하고 싶다
- 메뉴를 쉽게 탐색하고 원하는 것을 찾고 싶다
- 주문 상태를 확인하고 싶다
- 추가 주문을 편리하게 하고 싶다

### 동기 (Motivations)
- 직원을 부르지 않고 편하게 주문
- 메뉴 사진과 설명을 보고 신중하게 선택
- 현재까지 주문한 내역과 금액 확인

### 불편사항 (Pain Points)
- 복잡한 인터페이스나 로그인 절차
- 느린 반응 속도
- 주문이 제대로 들어갔는지 불확실한 상황
- 글씨가 작거나 버튼이 작아서 터치하기 어려운 UI

### 사용 시나리오
1. 테이블에 앉으면 태블릿이 이미 메뉴 화면을 보여줌
2. 카테고리를 탐색하며 메뉴 선택
3. 장바구니에 담고 수량 조절
4. 주문 확정 후 주문 번호 확인
5. 필요 시 추가 주문 또는 주문 내역 확인

## Persona 2: 매장 관리자 (Store Admin)

### 프로필
| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (대표 페르소나) |
| **역할** | 매장 운영자 / 사장님 |
| **연령대** | 30~50대 |
| **기술 수준** | 중급 (PC/태블릿 업무 사용 가능) |
| **사용 환경** | 카운터 PC 또는 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 들어오는 주문을 실시간으로 놓치지 않고 확인하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 추가/수정/삭제하고 싶다
- 테이블 회전율을 효율적으로 관리하고 싶다

### 동기 (Motivations)
- 인건비 절감 (주문 접수 인력 감소)
- 주문 누락 방지
- 매장 운영 효율성 증대
- 고객 대기 시간 감소로 만족도 향상

### 불편사항 (Pain Points)
- 바쁜 시간에 주문을 놓치는 상황
- 테이블별 주문 현황 파악이 어려운 상황
- 메뉴 변경 시 복잡한 절차
- 이전 고객 주문이 남아있어 혼란스러운 상황

### 사용 시나리오
1. 매장 오픈 시 관리자 로그인
2. 대시보드에서 테이블별 주문 현황 모니터링
3. 신규 주문 알림 확인 → 상태 변경 (준비중 → 완료)
4. 고객 퇴장 시 테이블 이용 완료 처리
5. 필요 시 메뉴 추가/수정, 과거 주문 내역 확인
