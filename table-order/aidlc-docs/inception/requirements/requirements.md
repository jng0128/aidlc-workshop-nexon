| 계층 | 기술 |
|------|------|
| **프론트엔드** | TypeScript + React |
| **백엔드** | TypeScript + NestJS |

---

## 2. 시스템 구성

### 2.1 아키텍처 개요
- **단일 매장** 운영 (멀티테넌시 불필요)
- **소규모** 운영: 1~10개 테이블, 동시 접속 10명 이하
- **모노레포** 구조: 프론트엔드(고객용 + 관리자용) + 백엔드 + DB

### 2.2 주요 구성 요소
- **고객용 웹 UI**: 테이블 태블릿에서 동작하는 브라우저 기반 주문 화면
- **관리자용 웹 UI**: 매장 운영자가 사용하는 관리 화면
- **백엔드 API 서버**: NestJS 기반 REST API + SSE
- **PostgreSQL 데이터베이스**: 매장, 메뉴, 주문 정보 저장

---

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지(외부 URL)
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공 시: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 비우기
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료/취소)
- 매장 이용 완료 처리된 주문 제외

### 3.2 관리자용 기능

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- 관리자 계정은 시드 데이터로 초기 생성

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료 (+ 취소 가능)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- **테이블 세션 종료(이용 완료)**: 확인 팝업 → 주문 내역 과거 이력 이동 → 현재 주문/총액 리셋
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록(시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 후 2초 이내 관리자 화면 표시
- 소규모 운영 기준 최적화 (동시 10명 이하)

### NFR-02: 가용성
- Docker Compose 기반 배포로 간편한 운영
- 브라우저 새로고침 시 세션/장바구니 유지

### NFR-03: 보안
- JWT 토큰 기반 인증
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- (보안 확장 규칙 미적용 - MVP 수준)

### NFR-04: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 한국어 기본 + 영어 지원 (i18n)
- 직관적인 카드 기반 레이아웃

### NFR-05: 유지보수성
- TypeScript 정적 타입 시스템
- 모노레포 구조로 코드 공유 용이
- Docker Compose로 개발/배포 환경 일관성

## 5. 주문 상태 흐름

```
대기중(PENDING) → 준비중(PREPARING) → 완료(COMPLETED)
       ↓                ↓
    취소(CANCELLED)   취소(CANCELLED)
```

- 대기중에서 취소 가능
- 준비중에서도 취소 가능
- 완료 상태에서는 취소 불가

## 6. 테이블 세션 라이프사이클

```
[새 고객 착석] → 첫 주문 시 세션 시작 → 추가 주문 가능 → 이용 완료 처리 → 세션 종료
                                                              ↓
                                                    주문 내역 → 과거 이력 이동
                                                    현재 주문/총액 → 0 리셋
```

## 7. 제외 범위 (Out of Scope)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 업로드/리사이징 (외부 URL만 사용)
- 알림 시스템 (푸시, SMS, 이메일)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석, 매출 리포트, 직원 관리, 예약, 리뷰)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)

## 8. 다국어 지원 (i18n)

- 기본 언어: 한국어

- 구현 방식: 프론트엔드 i18n 라이브러리 활용 (react-i18next 등)
- 대상: 고객용 인터페이스 (관리자 인터페이스는 한국어 전용)
